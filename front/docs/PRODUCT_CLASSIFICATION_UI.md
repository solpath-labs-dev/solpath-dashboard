# UI — `상품 항목 분류`

기존 대시보드(동기화 페이지) **같은 앱/스타일**에서 탭(또는 섹션)을 **연장**한다.

## 탭/페이지 제목

- **상품 항목 분류**

## 콘텐츠(이전에 합의한 레이아웃)

- **같은 화면** — 위: **미분류**, 아래: 솔패스 / 솔루틴 / 챌린지 / 교재 — 각 **한 행(접기)**, 펼치면 리스트
- (확장) 미분류만 필터, 분류마다 **수량 배지**, **검색**
- `internal_category`·`lifecycle`은 **드롭다운**으로 변경. **드롭다운을 바꾸자마자 DB에 쓰지 않는다** — 아래 "저장" 참고
- **미분류·이미 분류된 항목 모두** 드롭다운으로 수정 가능

## 저장 동작(실수 방지)

- 화면에서 드롭다운(및 기타 편집 필드)을 바꾸면 **로컬 상태만** 변경한다.
- **오른쪽 상단**에 **「수정하기」**(또는 "변경 사항 적용") 버튼을 두고, **다른 곳에 저장이 반영될 만큼 뭔가 바뀐 경우에만 활성화**한다.
- 사용자가 **「수정하기」를 누를 때만** GAS `POST`로 **일괄 반영(배치)**한다. (새로고침/탭 이탈 시 경고는 선택)

## 상품명 표시

- 리스트/카드에 보이는 상품명은 **최대 20자**로 잘라 표시(말줄임). (API `product_name_display`도 동일 길이)
- 20자를 넘는 경우 **전체 이름**은 마우스 **호버 시 툴팁**으로 보여준다. (일반 텍스트에는 HTML **`title` 속성** 권장.)

## 운영 DB(시트)가 **아직 없을 때**

- 이 페이지에 **처음** 들어왔을 때, 백엔드 `productMappingState`(명세: [../../docs/BACKEND_API.md](../../docs/BACKEND_API.md) §2.3)로 **스프레드시트·`product_mapping` 시트가 준비됐는지** 확인한다.
- **준비되지 않았으면** 본문 UI 대신 **「상품 불러오기」** 버튼(문구는 구현 시 확정) **만** 보인다.
- 그 버튼을 누르면 GAS `POST` **`initOperationsSheets`**(같은 절)를 호출해, **원천 DB가 있는 Drive 폴더**에 **새 스프레드시트**를 만들고 `SHEETS_OPERATIONS_ID`에 연결한 뒤, `product_mapping` 탭·헤더를 생성한다.
- 완료 후 **일반 UI**(상태 재조회 → 목록)로 진입한다.
